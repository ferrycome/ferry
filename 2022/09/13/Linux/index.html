Linux常用命令

linux基本命令

关机和重启

shutdown -h now //立刻关机

shutdown -h 5 //5分钟后关机

poweroff //立刻关机

shutdown -r now //立刻重启

shutdown -r 5 //5分钟后重启

reboot //立刻重启

帮助命令

–help //显示linux命令的帮助信息

补充：如果系统没有man命令，可以通过输入程序安装

"sudo apt-get install manpages-posix manpages-posix-dev"

GCC编译器

简介

GCC是以GPL许可证所发行的自由软件，也是 GNU计划的关键部分。GCC初衷是为GNU操作系统专门编写一款编译器，现已被大多数类Unix操作系统(如Linux、BSD、Mac OSx等）采纳为标准的编译器。

​ GCC原名为GNUC语言编辑器（GNU C Compiler)，只能处理c语言。但其很快扩展，变得可处理C++，后来又扩展为能够支持更多编程语言，如 Fortran、Pascal、Objective-C、Java、Ada、Go 以及各类处理器架构上的汇编语言等，所以改名GNu编译器套件（GNU Compiler Collection)。

编译过程

预处理

插入头文件，替换宏

gcc -E test.c -o test.i

编译

编译成汇编

gcc -S test.i -o test.s

汇编

编译成目标文件

gcc -c test.s -o test.o

链接

链接到库中，变成可执行文件

gcc test.o -o test

一次性生成

gcc test.c -o test

GCC生成动态库和静态库

单个源文件/目标直接生成动态库

gcc -fPIC -shared xxx.c -o libxxx.so
gcc -fPIC -shared xxx.o -o libxxx.so

多个源文件/目标直接生成动态库

gcc -fPIC -shared xxx1.c xxx2.c xxx3.c -o livxxx.so
gcc -fPIC -shared xxx1.o xxx2.o xxx3.o -o livxxx.so

单个源文件/目标直接生成静态库

ar -rc libxxx.a xxx.0

多个源文件/目标直接生成静态库

ar -rc libxxx.a xxx1.o xxx2.o xxx3.o

Makefile编写

简介

​ Makefile文件描述了整个工程的编译、链接等规则。其中包括:工程中的哪些源文件需要编译以及如何编译、需要创建哪些库文件以及如何创建这些库文件、如何在最后产生我们想要的可执行文件。尽管看起来可能是很复杂的事情，但是为工程编写Makefile的好处是能够使用一行命令来完成”自动化编译”，一旦提供一个(通常对于一个工程来说会是多个）正确的Makefile。编译整个工程所要做的事就是在shell提示符下输入make命令。整个工程完全自动编译，极大提高了效率。

Makefile规则

target...:prerequisites...
    command
    ...
    ...

target //可以是一个object file，也可以是一个课执行文件，还可以是一个标签

prerequisites //生成该target所依赖的文件和/或target

command //该target要执行的命令

例如

hello: hello.c hello.h
    gcc hello.c -o hello

make编译过程

(1）读入所有的Makefile文件;
(2）读入被include包括的其他Makefile文件;
(3）初始化文件中的变量;
(4）推导隐式规则，并分析所有规则;
(5）根据依赖关系，决定哪些目标要重新生成;

(6）执行生成命令。

Shell编程

简介

​ Shell是 linux中一个重要的层次，它是用户与系统交互作用的界面。在介绍linux命令时，shell都作为命令解释程序出现:它接收用户打入的命令，进行分析，创建子进程实现命令所规定的功能，等子进程终止工作后，发出提示符。这是shell最常见的使用方式。
​ Shell除了作为命令解释程序以外，还是一种高级程序设计语言，它有变量，关键字，有各种控制语句，如if，case，while，for等语句，有自己的语法结构。利用shell程序设计语言可以编写出功能很强、但代码简单的程序，特别是它把相关的linux命令有机地组合在一起，可大大提高编程的效率，充分利用linux系统的开放性能，设计出适合自己要求的命令。

Shell语法

Shell变量

用户定义的变量

变量名时以字母或下划线打头的字母、数字和下划线字符序列，并且大小写敏感

MYFILE=/usr/meng/ff/m1.c
位置参数

​ 执行Shell 脚本时可以使用参数。由出现在命令行上的位置确定的参数称做位置参数。在sh中总共有十个位置参数，其对应的名称依次是$0,$1,$2….9。其中$o始终表示命令名或Shell脚本名，对于一个命令行，必然有命令名，也就必定有$0。而其他位置参数可以根据需求可有可无。

特殊字符

通配符

* //匹配任意字符的0次或多次出现

? //匹配任意一个字符

[] //其中有一个字符组，匹配该字符组所限定的任意一个字符

引号

双引号 //除$、倒引号和反斜线仍保留其功能，其余字符作为普通字符处理

单引号 //所有字符都作为普通字符

倒引号 //用于执行shell命令

反斜线 //转义字符

GIT概念介绍

git对象模型

blob //用赖存储文件数据，通常是一个文件

tree //类似于一个目录，用来管理一些tree或是blob

commit //指向一个tree对象，用来表示某次提交时的状态，包括一些关于时间点的元数据(时间戳、最近一次提交的作者、指向上次提交的指针等)

tag //也成为里程碑，用来表示某个重要的commit，可以通过tag名称来找到这个commit

//用stable-1 作为提交[1b2e1d63ff]的代称
"git tag stable-1 1b2e1d63ff"
"git tag -a stable-1 1b2e1d6ff"

索引对象的方法

直接填入SHA-1码

"git log 42c4038c46249ce61ba68f99cb0a0e0c68f5a691
"

填入SHA-1码的前几位，保证没有重复即可

"git log 42c403"

访问某个提交的某个文件

HEAD //表示最近一次提交

HEAD^ //表示倒数第二次提交

HEAD~n //表示倒数第n次提交

"git show HEAD^:frivers/usb/dwc_otg/dwc_otg_driver.c"

双点语法 //获得一个分支有而另一个分支中没有的

//得到C、D
"git log master..experiment"

三点语法 //获得两个分支不共享的分支集合

//得到C、D、E、F
"git log master...experiment"

git config 参数设置

//设置用户信息
"git config --global user.name 'your name'"
"git config --global user.email 'your@email'"

git每次提交要求必须有相应的用户名和邮箱信息，加上–global参数会将用户信息写入到~/.gitconfig目录下，作为全局设置

//开启彩色ui效果
git config --global color.ui true

//设置差异比较工具为vim
git config --global diff.tool vimdiff

//不检查文件模式差别
git config --add cre.filemode false

git alise 命名别名

方法一：在~/.gitconfig文件中添加[alise]字段

[alias]
st = status
ci = commit
br = branch

方法二：用命令行的方式

"git config --global alise.st status"
